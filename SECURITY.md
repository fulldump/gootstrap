# Security Policy

## Supported Versions

Security fixes are provided for the latest released version on `master`.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, report privately using GitHub Security Advisories:

1. Go to the repository `Security` tab.
2. Click `Report a vulnerability`.
3. Include impact, affected versions, and reproduction details.

You can expect an initial response within 72 hours.

## Disclosure Process

- We will confirm receipt and assess severity.
- We will work on a fix and coordinate a release.
- We will publish an advisory when remediation is available.
